# Edge Flow / Lifted Valleys — Progress Log

Date: 2025-10-21

## Problem Description
The "lifted valleys" are being placed incorrectly: instead of forming an outward-bridging linear fill between adjacent ridge peaks (closing the valley), the algorithm sometimes creates a distinct bump inside the depression, not tied to the two neighboring peaks in the same ring. The goal is to ensure lifted valleys align with the true angular minima of the base superformula radius r_base(θ,z) between adjacent peaks and to apply an outward-only bridge B(θ,z) across each peak-to-peak sector so the final envelope is >= max(r_base, B).

## Goals
- Locate the true angular minima between each adjacent peak pair per ring (z).  
- Compute a straight linear bridge B(θ) between the two peak radii across the sector.  
- Apply changes only outward (no inward lowering) and only inside the peak-to-peak sector.  
- Keep the algorithm sector-local, mesh- and twist-independent, and consistent with existing spec.

## High-level solution approach
- Work in a de-rotated (twist-compensated) analysis frame for per-ring detection, then map results back to raw theta indices using the integer per-ring column shifts `s_tw`.
- For each adjacent peak pair (a,b) in a ring:
  - Choose the short angular arc between them (wrap-aware).
  - Build a periodic interpolant of r_base on that arc and fine-sample to find theta_val = argmin r_base(θ) on the arc.
  - Compute B(θ) as a linear interpolation between r_peak_a and r_peak_b across the sector discrete columns.
  - Apply Env_ext[zi, idxs] = max(Env_ext[zi, idxs], B_vals) (outward-only).
