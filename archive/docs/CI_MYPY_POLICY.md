# CI & per-module mypy tightening policy

This document describes the incremental mypy tightening process used by the
repository CI. It records the per-module approach, the auto-format policy, and
rules for detect-secrets baseline hygiene.

Goals
- Gradually re-enable strict mypy checks without producing large noisy PRs.
- Keep CI useful and deterministic; surface actionable type issues.
- Avoid accidental replacement of curated `detect-secrets.baseline` files.

Process
1. Pick a target module (small → large). Example order: `pfui` → `tools` → `potfoundry.core.geometry` → other core modules.
2. Create a branch `mypy/<module>-tighten` and a focused PR against the working base (e.g., `fix/edgeflow-debug-quick`).
3. Add or update a per-module mypy section in `mypy_ci.ini`, enabling one or more stricter flags (for example:
   - `disallow_untyped_defs = True`
   - `disallow_incomplete_defs = True`
   - `warn_unused_ignores = True`
   - `warn_return_any = True`
   - `warn_no_return = True`
   Start with a small set and expand as the PRs progress.)
4. Run checks locally:
   - `python -m mypy --config-file mypy_ci.ini <module-or-file>`
   - `pre-commit run --files $(git ls-files <module-path>)`
   - `pytest -q` (or the project's fast marker)
5. Fix issues with minimal, low-risk edits (add annotations, narrow Any uses, add TYPE_CHECKING guards). Prefer small commits.
6. Push the branch and open the PR. CI will run `pr-validation-full.yml`. Download artifacts and iterate on remaining failures.
7. Once all module PRs are merged, remove per-module relaxations and adopt the strict `mypy.ini` baseline in CI.

Auto-format policy
- CI offers an `auto-format` job that runs only safe formatting hooks (ruff, ruff-format, end-of-file-fixer, trailing-whitespace).
- The job will attempt to commit fixes. To avoid a detached-HEAD push failure, contributors should run the formatting hooks locally and commit before pushing. Example commands:

```powershell
# PowerShell (Windows)
pre-commit run ruff --all-files
pre-commit run ruff-format --all-files
pre-commit run end-of-file-fixer --all-files
pre-commit run trailing-whitespace --all-files
git add -A
git commit -m "style: apply formatting fixes"
git push
```

Detect-secrets baseline hygiene
- Keep the curated `detect-secrets.baseline` under source control. Do not overwrite it with an autogenerated baseline from CI.
- Add `--exclude-files` rules in `.pre-commit-config.yaml` to ignore generated CI artifacts, temporary baselines, and third-party caches to reduce noise.
- For any new true positives (real secrets), follow the security workflow: revoke/rotate the secret and add the token to the baseline only after verification.

Rollback plan
- If a module-wide tightening causes broad regressions, revert the per-module flags in `mypy_ci.ini` and open targeted follow-up PRs for broken areas.

Contact
- For coordination or large module changes, open an issue and assign reviewers. Keep PRs small and focused.
