## Load-bearing facts

1. **The shaped-template gate** (`QuadtreeTriangulator.ts:~121`): `shapedTemplate(efg, du, dt, uBias)` → true iff `efg` present AND (`uBias > 0` OR `cellAspect3D(efg,du,dt) > ANISO_ASPECT_GATE`), where `ANISO_ASPECT_GATE = Math.sqrt(3) * (1 + 1e-6)` — an isotropic cell falls through to legacy byte-for-byte.
2. **TWO transition branches route through earClip**: singleMid (~:510-536) AND N-mid (~:540-564), each with an `aniso && efg` arm (tags EAR_CLIP) vs centroid-fan arm (TRANSITION_FAN). The plain-quad branch (~:480-507) has the shorter-3D-diagonal choice via `metricLen2` with QSCALE tie-quantization (tie → SW→NE → byte-identical legacy).
3. **earClipMaxMinAngle (~:170-212) retains both crack hazards** despite its optimistic doc comment: collinear ears are `continue`-skipped (not scored −∞); if ALL remaining ears are collinear → `bestPrev<0` → `break` → stranded boundary sub-edges; the final-3 emission has no area check. Klincsek DP replacement remains justified (certified k−2 coverage).
4. **`triMinAngle3D(efg, p0, p1, p2)`** (~:128-148): law-of-cosines on `metricLen2` sides — the metric scorer the DP reuses as its weight function. `metricLen2(efg,du,dt) = E·du² + 2F·du·dt + G·dt²`.
5. **Warps are piecewise-LINEAR with NO exported derivative.** `UWarp = {isIdentity, anchors: {source,target}[]}` (periodic circle homeomorphism, seam fixed 0→0/1→1); `TWarp` same but interval with fixed endpoints. `applyUWarp` (CreaseUWarp.ts:~214-234) / `applyTWarp` (CreaseTWarp.ts:~229-245) do linear interpolation over `[0,...anchors,1]`. **PullbackMetric needs a new `uWarpDerivative(warp, u)` / `tWarpDerivative(warp, t)` = the containing segment's slope (piecewise-constant, discontinuous at anchors/kinks)** — the spec's "warp-anchor kinks lie on cell boundaries" assertion matters here. `chooseCreaseGrid`/`chooseCreaseTGrid` snap anchors onto power-of-two lattices (sources are dyadic, level 3..6) → kinks ARE on dyadic cell boundaries by construction (verify in plan with an assertion).
6. **Warp application guards** (ParametricExportComputer.ts ~:2509-2539 + helix ~:2540+):
   - u-warp: `!creaseChoice.warp.isIdentity` → applied to **ALL surfaces** (u is angular everywhere): `asm.vertices[i] = applyUWarp(creaseChoice.warp, asm.vertices[i])`.
   - t-warp: `!creaseTChoice.warp.isIdentity` → **walls only** (`surfaceId < 1.5`); caps reuse t as radial.
   - helix: `!helixChoice.warp.isIdentity && creaseChoice.warp.isIdentity` (helix XOR u-warp); walls sheared by own t, rim cap tEval=1, base caps tEval=0.
   - **Composed outer-wall map** for the metric: `(u,t) ↦ P(applyHelix(helix, applyUWarp(uw, u), t'), applyTWarp(tw, t))`… exact composition ORDER must mirror the application code: u-warp first (all vertices), then t-warp (walls), then helix (reads the ALREADY-u-warped u; helix only fires when u-warp is identity, so in practice u-warp and helix never compose). Effective per-style outer-wall map = `P(applyHelixWarp(helix, u, ψ(t)) ∘ φ(u), ψ(t))` with the XOR guard collapsing it to one of: plain / φ-only / ψ-only / φ+ψ / helix-only / helix+ψ.
7. **Metric threading point**: `buildQuadtreeAtScale` (ConformingWall.ts:~203-230) constructs `new PeriodicBalancedQuadtree(field, sampler, {maxLevel, pinBoundaryLevel, minUniformLevel, featureRefine, creaseRefine, uBias, directionalRefine})` — the 2nd positional arg IS the metric sampler (currently the plain wall sampler). Threading a warp-composed metric sampler = new optional `ConformingWallOptions.metricSampler` → pass as the 2nd arg (sizing `field` stays on the PLAIN sampler per the spec). `AssemblyWallOptions` (WatertightAssembly.ts:~174-247, full interface extracted) gains the per-wall metric samplers; call sites at ~:444-452 (outer gets featureLines/creaseLines; inner plain).
8. **efg population site**: `leaves()` (PeriodicBalancedQuadtree.ts:~554-565) maps cells → QuadLeaf literals; `leafOfCell` (~:583) is the hot neighbor path (do NOT compute efg there). `QuadLeaf.efg?` is declared (~:65) and never assigned in production. `firstFundamentalForm(metric, uc, tc, hu, ht)` (SurfaceMetricTensor.ts:~115-126) + `metricStepsForSampler(s, cells=1)` (~:33-46; falls back to `DEFAULT_H=1e-4` when no `gridResolution()` — the composed wrapper must FORWARD gridResolution from the plain sampler).
9. **Test infrastructure for efg ALREADY EXISTS**: `QuadtreeTriangulator.test.ts` (~481 lines) has `ScaledPlaneSampler` (E=SX²,F=0,G=SY²), `ShearedPlaneSampler` (E=SX², F=SX·SHEAR, G=SHEAR²+SY²) with `.efg()` helper, `to3D`/`nonSeamIndices`/`minAngle3D`/`degenerate3D` helpers, hand-built `QuadtreeLike` fixtures (`{leaves: () => leaves, uBias: () => 0}`), and a byte-identity test (isotropic efg ⇒ indices+vertices identical to no-efg legacy). It is the ONLY test file referencing efg. Klincsek property tests extend THIS file's idiom.
10. **FCT mirror points**: `FeatureTriangulationOptions` ( :45-68: quantScale, cornerSnap, sampler) — gains the efg/metric input for its plain branch ( :703-724: FCT_PLAIN_QUAD 0-split arm + FCT_PLAIN_FAN centroid arm; no other fan emission in the feature path).
